# 📊 周报自动生成智能体 (WeeklyReportAgent)

> 基于 AI 的智能周报助手，自动整理工作内容并生成结构化周报

## 🎯 项目简介

本项目是一个 **AI 智能体应用系统**，专注于帮助职场人士和学生组织成员高效完成周报撰写工作。用户只需输入一周的工作事项，系统即可自动分类整理并生成规范的结构化周报，同时支持将周报归档到飞书多维表格。

## 📋 功能特性

| 功能 | 说明 |
|------|------|
| 📝 工作事项收集 | 支持多行输入，每行一个工作事项 |
| 🗂️ 智能分类整理 | 自动按日常事务、项目任务、沟通协作、学习成长等类别整理 |
| 📊 周报生成 | 输出结构化 Markdown 格式周报 |
| ☁️ 飞书集成 | 支持创建多维表格并保存周报数据 |
| 💬 多轮对话 | 支持补充姓名、日期、下周计划等信息 |
| 🧠 短期记忆 | 自动保留最近对话历史 |

## 🚀 快速开始

### 环境要求

- Python 3.10+
- uv 包管理器

### 安装依赖

```bash
cd /path/to/project
uv sync
```

### 配置说明

编辑 `config/agent_llm_config.json` 文件，配置模型参数：

```json
{
    "config": {
        "model": "doubao-seed-2-0-lite-260215",
        "temperature": 0.7,
        "top_p": 0.9,
        "max_completion_tokens": 10000,
        "timeout": 600,
        "thinking": "disabled"
    },
    "sp": "# 系统提示词...",
    "tools": ["format_work_items", "generate_markdown_report", "..."]
}
```

### 运行 Agent

```bash
uv run python -m src.main
```

### 使用示例

```
用户: 请帮我整理本周的工作内容并生成周报：
```

## 🛠️ 工具函数说明

| 工具函数 | 功能说明 | 输入参数 |
|----------|----------|----------|
| `format_work_items` | 整理和分类工作事项 | work_items: str |
| `generate_markdown_report` | 生成 Markdown 周报 | user_name, week_date, work_summary, next_week_plan |
| `create_feishu_weekly_table` | 创建飞书多维表格 | table_name |
| `save_weekly_to_feishu` | 保存周报到飞书 | app_token, table_id, user_name, week_date, work_items |

## 📖 设计文档

### 服务对象

- 职场人士（定期撰写周报的上班族）
- 学生组织成员（社团、项目组的周总结需求）
- 项目团队（敏捷迭代的周报记录）

### 具体问题

- 周报撰写费时费力
- 工作事项遗漏
- 分类整理不规范
- 周报格式不统一

### 输入输出

| 输入 | 输出 |
|------|------|
| 工作事项（多行文本） | 分类整理后的工作列表 |
| 姓名、日期范围 | 结构化 Markdown 周报 |
| app_token、table_id | 飞书多维表格记录 |

### 处理步骤

1. 收集用户输入的工作事项
2. 调用 `format_work_items` 进行智能分类
3. 询问用户补充信息（姓名、日期等）
4. 调用 `generate_markdown_report` 生成周报
5. 询问用户是否需要归档到飞书
6. （可选）调用 `create_feishu_weekly_table` 创建表格
7. （可选）调用 `save_weekly_to_feishu` 保存数据

### 异常处理

- 输入为空时：提示用户输入工作事项
- 信息不完整时：主动询问补充
- 飞书接口失败时：返回友好错误提示

```python
    """工具描述"""
```
